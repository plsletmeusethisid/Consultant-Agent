Q 3.4: “Internal Authentication Error” – Cato VPN시스템 오류

“Internal Authentication error”은 다양한 원인 (f이메일 / 패스워드 오입력, 시스템 반영 지연 등)으로 인해 뜰수 있습니다. 정확히 어떠한 원인때문에 일어난다고 단정할수 없습니다.
하지만, “Internal Authentication error”가 뜰 경우, 사용자님한테 Reset Password 혹은 초대 메일을 보내서 비밀번호를 재설정할수 있도록 하면 됩니다.
비밀번호 변경 후 시스템 반영 지연 시간이 지난 이후에도 동일한 에러 발생 시, Cato Client의 Users 탭에서 사용자 계정을 잠시 등록 해제한 후 다시 등록하는 방법을 시도합니다.
위에 내용을 진행해도 해결되지 않는다면 원격접속을 통해 와이파이, 이더넷, Cato 어댑터의 IPv6를 비활성화 해보시기 바랍니다. Windows+R, ncpa.cpl, 연결된 네트워크 우클릭, 속성, IPv6 끄기
위에 사항을 진행해도 안된다면 Cato Client > Settings에서 Authenticate with external browser 선택해서 웹페이지를 통해 로그인 시도
Reset Password 설정 후, 아래 myvpn.catonetworks.com으로 리다이렉션 됨. 해당 로그인은 진행 불필요.

Q 3.6: “Internal Cato error” - 비밀번호/MFA 설정 오류, internal PoP 오류
비밀번호, 혹은 MFA가 잘못 설정되었거나 미설정되어 나타나는 오류일 수 있으니, 사용자 메일로 reset password, reset MFA, 혹은 send invitation mail을 전송하여 다시 설정한다.
Password와 MFA 재설정을 반영 시간 기다리지 않고 이어서 바로 할 경우 발생할 수 있다. 
기다리면 풀릴 수 있으나, 10분 지나서도 해결 되지 않으면 재설정 다시 진행. 
위 절차 이후에도 문제가 해결되지 않는다면 Client 버전 확인 후, 최신 버전으로 업데이트 후 다시  MFA 재설정 후 로그인 시도
비밀번호/MFA 재설정 이후에도 지속적으로 오류 발생 시, log collect 하여 벤더측에 티켓팅 진행.
Q 3.7: Cato Client 연결 시 인터넷 접속이 불안정한 경우 - Client가 Connect, Disconnect를 반복
Cato Client 버전 확인, 타 보안프로그램 설치 여부 확인, Cato Client 재설치 및 단말 재부팅 이후에도 계속 인터넷 접속에 문제가 발생하는 경우, Cato PoP 연결에 문제가 있는 것일 수 있음.
따라서 Cato PoP을 Manual하게 지정한 후 인터넷 접속 재시도.
예) Seoul PoP (140.82.205.1 - 140.82.205.254)의 경우, 140.82.205.1에 접속하는 사용자가 많을 수 있음. 따라서 범위 내의 다른 IP주소로 지정해준 뒤 연결 재시도.

Q 5.1: “SSL Handshake error” - 발생하는 경우:
SSL 핸드셰이크(SSL handshake)는 클라이언트와 서버 간의 보안 통신을 설정하기 위한 프로세스. 
클라이언트와 서버 간에 안전한 연결을 설정할 수 없을 때 발생되며 이는 여러 이유로 인해 발생할 수 있기 때문에 드라이버 재설치등 설정과 관련하여 확인 필요.
